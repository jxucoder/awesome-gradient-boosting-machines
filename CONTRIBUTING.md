#### For Research Papers

1. Choose the appropriate year section (or create one if needed)
2. Follow this format:

```markdown
- **Paper Title** (Conference/Journal Year)
  - Author1, Author2, Author3
  - [[Paper](link-to-paper)] [[Code](link-to-code)]
```

3. Include:
   - Full paper title in bold
   - Conference/journal name and year in parentheses
   - All authors (use "et al." for more than 6 authors)
   - Link to paper (preferably arXiv or official publication)
   - Link to code implementation (if available)
4. Maintain chronological order within each year section
5. Place papers alphabetically within the same year

#### For Implementations & Tools

- Include the project name and link
- Add a brief description (one line)
- Mention star count for GitHub repos (e.g., ‚≠ê 10k+)

#### For Tutorials, Blog Posts & Videos

- Include the title and link
- Add author/source information
- Ensure the content is still accessible and up-to-date

### Suggesting Improvements

Have an idea to improve the list structure or organization? Open an issue first to discuss your proposal before submitting a PR.

## Pull Request Process

1. **Fork** the repository
2. **Create** a new branch: `git checkout -b add-paper-name` or `git checkout -b add-resource-type`
3. **Make** your changes
4. **Commit** with a clear message: `Add "Paper Title" to 2024 papers`
5. **Push** to your fork and **submit a Pull Request**

## What We're Looking For

### Papers We Accept

- Published in peer-reviewed conferences/journals
- Related to gradient boosting, adaptive boosting, or ensemble tree methods
- Include novel contributions (theory, algorithms, applications)

### Resources We Accept

- Official documentation and tutorials
- High-quality blog posts with substantial content
- Conference talks and educational videos
- Actively maintained tools and libraries

### What We Don't Accept

- Self-promotional content without substantial value
- Outdated resources (broken links, deprecated tools)
- Duplicate entries
- Low-quality or spam content
